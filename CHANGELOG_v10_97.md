# CHANGELOG v10.97 — Armed L1 Snapshot Breakout Retention

## Fixed
- 修復真實 runtime 長期無法完成「到中線 → ARMED_L1 → 後續突破 → 開 L1」的設計漏洞。
- 現價觸碰 / 穿越密集區中線時，只會啟動追蹤買入，並保留當刻密集區上下沿作為本輪突破觸發線。
- 已經 `ARMED_L1` 的幣，即使跌出排行榜前四，仍保留一個執行槽位；排行榜只補剩餘槽位，總執行計畫仍不超過四個。
- 已武裝後，即使六線/密集區後續重新計算、最新上下沿移動，L1 開倉判斷仍使用武裝當刻的保留上下沿，避免價格永遠追不上移動邊界。
- 已武裝後，如果市場快取暫時沒有密集區資料，不再抹走 `ARMED_L1` / `zone_entered`；狀態改為「保留追蹤」。
- 開 L1 時，實際 L1 價、止損邊界與階梯計算同步使用武裝快照上下沿。
- 新增真實時序 regression：真實候選幣先觸及中線、排行榜/密集區刷新後再突破，仍能按保留邊界開 L1。

## Unchanged
- 中線觸碰本身仍不買入，只啟動追蹤。
- 真正買入仍必須等已武裝後，現價突破密集區上沿或下沿。
- 成交量分不改。
- 波動分不改。
- 回測分不改。
- L1-L10 階梯不改。
- 止損 / 保盈公式不改。
- 排行榜權重不改。
- 真實模式守門、read-only/safe/no-real-orders 防護不改。

## Safety
- 未自動重啟交易。
- 未切換真實模式。
